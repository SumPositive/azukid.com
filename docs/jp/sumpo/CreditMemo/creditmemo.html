クレメモ　取扱説明

後日引き落とし記録アプリ

クレメモは、あとで口座から引き落とされる決済を記録して管理するためのアプリです。クレジットカード利用、クレジット連携の電子マネーチャージ、N日後引き落とし、指定日引き落とし などを管理できます。デビット払いや現金チャージのように、即時に引き落とされる決済は対象外です。

基本操作の流れ

1 決済を記録する

金額を入力し、必要に応じてラベルやタグを付けて保存します。

2 決済手段を選ぶ

カードや後払い手段を選び、引き落とし予定を決めます。

3 引き落としを確認する

未払と支払済みを確認し、口座ごとの予定を把握します。

4 見直し・保管する

履歴を修正したり、JSON でバックアップしたりできます。

⚠️ データは暗号化しないので、個人情報や秘密情報を入れないでください。
App Store エンドユーザー使用許諾契約 プライバシーポリシー 作者自己紹介: sumpo

できること

音声で新しい決済金額とラベルを話すだけで明細を作成。続けて決済手段も発話で指定できます。
Siri で音声入力「Hey Siri クレメモで音声入力」と話すだけで、アプリを開いて音声入力を開始できます。
配色プリセット 12 種引き落とし状況とメニューアイコンの配色を 12 種類から選べます。アプリアイコンにも反映されます。
新しい決済をすばやく入力金額を先に入れて保存し、細かい情報はあとから補えます。
履歴をあとから修正保存後の明細を開いて、ラベル・タグ・決済手段・口座などを更新できます。
引き落とし予定をまとめて確認直近の引き落とし合計と、未払・済みの状態を一覧できます。
口座別に集計同じ引き落とし日でも、口座ごとに分けて把握できます。
決済手段ごとに支払日を管理締日/支払日型と N日後型に対応します。
タグを複数付与達人モードではタグ別の整理や再編集ができます。
JSON でエクスポート/インポートバックアップや他端末への移行に使えます。
表示や保存後動作を調整文字サイズ、起動時画面、保存後の戻り先などを設定できます。

画面構成

音声で新しい決済 マイクで明細を作成します。最初に金額とラベルを話し、必要に応じて決済手段も話して指定できます。検出した手段はタップで一覧から選び直せます。「Hey Siri クレメモで音声入力」と話して起動することもできます。
新しい決済 新規の決済明細を入力します。起動時にこの画面を開く設定もできます。
履歴 保存済みの明細を一覧します。対象期間、未入力あり、決済手段、口座、タグで絞り込み、日付や金額で並べ替えできます。
引き落とし状況 引き落とし前と引き落とし済みをまとめて表示します。日付、決済手段、口座の集計を切り替え、手段や口座で絞り込めます。
決済手段 カード名や後払い手段、締日/支払日、引き落とし口座を登録します。
引き落とし口座 引き落とし先の口座名を管理します。決済手段ごとに関連付けできます。
タグ 用途や分類のためのタグです。
設定 表示、保存後動作、バックアップ、サポート情報をまとめています。

新しい決済を入力する

まずは 金額を入れて保存 してください。決済手段、ラベル、タグはあとから補完できます。初心者モードではこの流れに沿った案内を表示します。

基本入力

金額 専用テンキーで入力します。保存に必須です。
利用日 カレンダーから選択します。引き落とし予定日は、決済手段の設定に基づいて自動計算されます。
決済手段 カードや後払い手段を選びます。未選択のまま保存し、あとで履歴から補うこともできます。
引き落とし口座 選んだ決済手段に口座が未設定のときだけ、この画面で口座を選べます。
ラベル 店名や用途など、一覧で見分けやすい短い名前を入れます。過去入力から候補も表示されます。

補足入力

タグ 達人モードで複数選択できます。あとで変更や追加もできます。
繰り返し 達人モードの一括払いで使えます。対象の決済を引き落とし済みにしたとき、翌月など指定した間隔の新しい決済を自動追加します。
メモ 自由記入欄です。あとから見返したい補足情報を残せます。

似た決済の流用

既存明細からコピー 決済一覧で金額一致の明細・最近編集した明細を一覧から選んでコピー、決済手段一覧の明細をコピー、引き落とし明細の引き落とし日＋決済手段をコピー、を新規決済に反映できます。

引き落とし日（支払日）の自動／手動

自動 利用日と決済手段（締日／支払日／支払月／N日後型）から引き落とし日を自動計算します。利用日や決済手段を変更すると自動的に追従します。日本ロケールで設定 ON のときは、土日祝に当たる場合は翌営業日へ繰延します。
手動 アイコンをタップすると、その時点の日付に固定されます。以降は利用日や決済手段を変えても引き落とし日は変わりません。締切に間に合わなかった決済を翌回引き落としへずらしたい場合などに使います。日付タップでカレンダー選択、または「前支払日へ／翌支払日へ」ボタンで支払サイクル単位の移動ができます。
済みで手動に固定 明細を「引き落とし済み」にすると、その明細の引き落とし日は自動的に「手動」に切り替わって固定されます（誤って利用日や決済手段の変更で動かないようにするため）。未払に戻しても固定は保持されますが、アイコンをタップすれば「自動」に戻せます。
2回払い 引き落とし日が 1回目・2回目の 2 行で表示され、自動／手動のアイコンは 2 行まとめて切り替わります。分割金額は自動で配分されますが、金額をタップすれば 1円〜総額-1円 の範囲で手動配分でき、もう一方に振り分けられます。2回目の支払日は 1回目の翌日以降に制限され、1回目を変更して 2回目以降になった場合は 2回目を翌月へ自動補正します。1回目を「引き落とし済み」にすると、2回目の引き落とし日以外（金額・利用日・決済手段など）は編集不可になります。設定の「2回払いを利用する」を ON にすると新規入力で選べるようになります（既存の 2回払いは設定 OFF でも表示・編集を維持）。

履歴を見直す

並び順 日付または金額で並べ替えできます。それぞれ昇順/降順を切り替えられます。
フィルター 対象期間は 3ヶ月、1年、3年、全期間から選べます。さらに、すべて、未入力あり、決済手段、口座、タグ複数選択で絞り込めます。未入力ありは、決済手段未設定・決済ラベル未入力の明細です。
編集 一覧の明細を開くと、内容の修正や削除ができます。すでに引き落とし済みの明細は、金額・利用日・決済手段など一部の中核項目が固定されます。

引き落とし状況

引き落とし状況では、未払の予定と引き落とし済みを 1 つの流れで確認できます。引き落としを確認したら、矢印アイコンでその口座を 引き落とし済み に切り替えてください。必要なら未払へ戻すこともできます。
未払の区分 当日以降の予定と、引き落とし日を過ぎた未払を分けて表示します。過去未払があるときは切り替えセグメントが表示されます。
直近合計 トップ画面には、設定した期間ぶんの直近引き落とし合計を表示します。期間は 1〜20 日または 1 か月で変更できます。
口座別集計 同じ日付でも口座が異なる請求は別の引き落としとして集計します。集計は日付、決済手段、口座で切り替えられます。
絞り込み すべて、決済手段、口座から選べます。手段や口座を指定すると、時系列で確認しやすいよう日付集計へ切り替わります。
明細表示 引き落とし行を開くと、その引き落としに含まれる決済手段別の明細を確認できます。
明細ごとの調整 引き落とし明細では、明細単位で未払/済みを切り替えられます。締切に間に合わず次回に引き落とされる明細だけを移す時に使います。
ロックトグル 引き落とし明細では、明細ごとにロックできます。旧アプリの確認チェック機能を、チェックマークではなくロックマークとして実装したものです。通帳や明細書と照合済みであることを残すために使います。

決済手段・口座・タグ

決済手段

用途 後日口座から引き落とされる決済手段だけを登録します。PayPay ならクレジット連携利用や残高チャージが対象で、チャージ済み残高の支払いは対象外です。
請求方式 締日/支払日型と N日後型を選べます。締日が月末の場合は月末指定もできます。
口座関連付け 決済手段ごとに引き落とし口座を 1 つ設定します。変更すると、その決済手段に属する過去の請求や引き落としの所属も再計算され、古い口座側に残らないよう整理されます。
一覧から編集 決済手段一覧で行を選ぶと、その決済手段の編集画面を開きます。状況確認は引き落とし状況の集計・絞り込みで行います。

口座・タグ

口座 引き落とし先の口座名を管理します。プリセットから名前を引用することもできます。
タグ タグ編集画面では、そのタグが付いた決済明細を一覧で見返せます。

バックアップと設定

JSON バックアップ

エクスポート 全データを JSON ファイルとして書き出せます。保存向けのコンパクト形式と、人が読みやすいプリティ形式を選べます。
インポート 書き出した JSON を読み込めます。同じデータを再読込しても、置換ではなく重複しない形で更新します。請求と引き落とし状態も復元します。
古い履歴の整理 データ量が増えすぎた場合は、設定から古い決済を整理できます。運用上は 3 年程度を目安に保つ想定です。

表示と操作

初心者/達人 初心者は説明を多めに表示します。
文字サイズ 自動にすると iOSの文字サイズ設定に追従します。
起動時の動作 アプリ復帰時に「新しい決済」を自動で開く設定ができます。
テンキー自動表示 「新しい決済」を開いた時に金額が未入力なら、テンキーを自動で開きます。設定で ON/OFF を切り替えできます。
音声入力を使う メインメニューに「音声で新しい決済」を表示するかを切り替えます。端末ロケールが音声認識に対応している場合のみ表示されます（既定 ON）。
配色プリセット 引き落とし状況（未払／確認待ち／済み）とメニューアイコンの配色を 12 種類のプリセットから選べます。選んだプリセットはアプリアイコンにも反映され、ホーム画面の見た目も切り替えられます。
保存後の動作 前画面に戻る、連続入力する、同じ日付と決済手段を保って連続入力する、履歴を表示する、から選べます。
取扱説明と応援 設定から取扱説明をアプリ内で開けます。開発者を応援する項目では投げ銭や動画広告の視聴を選べます。

基本仕様

対象 後日口座から引き落とされる決済（即時引き落としのデビット、現金チャージは対象外）
対応 OS iOS 18 以降
金額上限 99,999,999
利用日範囲 2000/1/1 〜 2100/12/31
ラベル文字数 50 文字まで
メモ文字数 200 文字まで
支払方法 一括／2回払い（無利息・手数料無料の範囲を想定）。3回以上の分割やボーナス払いは未対応
引き落とし日計算 締日／支払月／支払日から自動計算。N日後型カードは利用日＋N日。決済手段未選択時は利用日と同日
土日祝の繰延 日本のロケールのみ、設定 ON（既定）で締日／支払日型のカードについて土日祝なら翌営業日へ繰延。1948年法施行〜2099年までの祝日に対応（特例措置・振替休日・国民の休日を含む）
引き落とし日のロック 明細単位で手動指定／自動計算を切替。済みにした明細は自動的にロックされる
繰り返し 1〜99 ヶ月の間隔で「済みにした時に翌月以降のコピーを自動生成」
タグ 1 明細に複数タグ。一覧の絞り込み・集計に利用
引き落とし合計期間 直近 1〜20 日、30 日、35 日、40 日、50 日、60 日（既定 15 日）
データ保存 端末ローカル（SwiftData）。クラウド同期なし。iCloud バックアップで端末ごと退避は可能
バックアップ／復元 設定画面から全データ JSON のエキスポート／インポート（コンパクト形式・プリティ形式）
JSON インポート方針 追記マージ（id 単位）方式。マスタ（口座・決済手段・タグ）／明細／状態（未払・済み）／引き落とし日ロック を保持。配列キーが欠けている場合は未指定として無視。途中で失敗した場合は変更を巻き戻して安全に終了
JSON インポート正規化 支払方法は 1（一括）または 2（2回払い）以外を一括へ正規化。N日後型カードの支払月は 0 に正規化。2回払いで手動配分した金額は復元時に保持。金額の解析に失敗した値は 0 として取り込み警告ログを出力
旧アプリ移行 旧 Core Data ファイル（AzCredit.sqlite）から自動移行。失敗時は次回起動で自動再試行。iCloud 復元で WAL/SHM が欠けても自動補修
古い履歴の整理 設定から指定年数より古い明細を一括削除可能
プライバシー 外部サーバーへの送信なし。ローカル DB は暗号化していないため、機微情報は記録しない設計

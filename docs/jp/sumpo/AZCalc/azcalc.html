AZCalc

BCD 十進演算と数式評価（iOS / macOS）

浮動小数点誤差のない BCD（Binary Coded Decimal）演算型 AZDecimal と、中置記法の数式文字列を評価する AZFormula を一つのパッケージで提供します。整数部・小数部それぞれ最大 30 桁（合計 60 桁）の精度を持ち、7 種の丸めモードに対応しています。

このパッケージの中核にある sbcd ライブラリは 1998 年に C/C++ で開発され、カルメモ（CalcRoll）創期から組み込まれてきた演算エンジンです。浮動小数点演算の誤差が許されない用途にご利用ください。2026 年に Swift 6 対応の Swift Package として独立公開しました。

GitHub Swift Package Index

デモ

特長

浮動小数点誤差なしBCD（2進化10進数）演算で 0.1+0.2=0.3 が保証される
60 桁の精度整数部・小数部それぞれ最大 30 桁。SBCD_PRECISION（Sources/AZDecimalC/include/SBCD.h）は #ifndef ガード付きで定義されており、パッケージをフォークしてビルドフラグ（-DSBCD_PRECISION=120）で変更可能。値は偶数に限る。上限は固定サイズ C 配列（char digit[SBCD_PRECISION+1]）のスタック消費により制限されます。
7 種の丸めモード四捨五入・銀行家の丸め・切り上げ・切り捨てなど
数式評価エンジンShunting Yard アルゴリズムによる中置記法の解析
% 演算子の文脈依存展開100+5% → 105（増量）/ 100×5% → 5（比率）
桁区切りフォーマット3桁・4桁・インド式・任意セパレータに対応
Swift 6 Sendable 対応struct で値型。スレッドをまたいで安全に渡せる
デモアプリ付属SwiftUI デモ（iOS 17+）が同梱。Xcode ですぐに試せる

AZDecimal

基本演算

import AZDecimal

let a: AZDecimal = "1.1"
let b: AZDecimal = "2.0456"
print(a + b)   // "3.1456"
print(a - b)   // "-0.9456"
print(a * b)   // "2.25016"
print(a / b)   // "0.537815..."

var x: AZDecimal = "10"
x += "3"   // 13
x -= "5"   // 8
x *= "2"   // 16
x /= "4"   // 4

便利プロパティ

AZDecimal.zero        // AZDecimal("0")
AZDecimal.one         // AZDecimal("1")

let v: AZDecimal = "-3.14"
v.isZero              // false
v.isNegative          // true
v.abs                 // AZDecimal("3.14")

丸め

// フルエント設定
let config = AZDecimalConfig.default
    .digits(2)
    .rounding(.r54)
    .trailingZero(true)
    .grouping(.threes)

let result = AZDecimal("3.456").rounded(config)
// AZDecimal("3.46")

// 書式化（丸め後に呼ぶ）
let value = AZDecimal("1234567.045")
print(value.rounded(config).formatted(config))
// "1,234,567.05"

丸めモード

モード名称説明規格
.rup切り上げ絶対値の大きい方向に丸める—
.rPlus正方向丸め正の無限大方向に丸めるIEEE 754: roundTowardPositive
.r54四捨五入5以上を切り上げJIS Z 8401 規則B
.r55五捨五超入偶数丸め・銀行家の丸めJIS Z 8401 規則A · IEEE 754: roundTiesToEven
.r65五捨六入5は切り捨て・6以上で切り上げ—
.rMinus負方向丸め負の無限大方向に丸めるIEEE 754: roundTowardNegative
.truncate切り捨て丸めなし・生の値を返すIEEE 754: roundTowardZero

フルエント設定（AZDecimalConfig）

メソッド説明
.digits(_ n: Int)小数桁数を設定
.rounding(_ type: RoundType)丸めモードを設定
.trailingZero(_ enabled: Bool)末尾ゼロの補充 / 削除
.grouping(_ type: GroupType, separator: String)桁区切りを設定
.decimalSep(_ separator: String)小数点記号を設定

AZFormula

基本的な使い方

import AZFormula

// 文字列を返す
switch AZFormula.evaluate("(100 + 5%) × 1.08") {
case .success(let value): print(value)  // "113.4"
case .failure(let error): print(error)
}

// 設定を指定して丸める
let config = AZDecimalConfig.default.digits(2).rounding(.r54)
let result = AZFormula.evaluate("1 ÷ 3", config: config)
// .success("0.33")

AZDecimal を返す

結果をさらに演算したい場合は evaluateDecimal を使います。

if case .success(let a) = AZFormula.evaluateDecimal("10+5"),
   case .success(let b) = AZFormula.evaluateDecimal("2+1") {
    print(a * b)   // AZDecimal("45")
    print(a.isZero)  // false
}

対応演算子

演算子説明
+ - × ÷四則演算（* / も受け付ける）
√平方根
∛立方根
( )括弧
%パーセント（文脈依存、下表参照）
割 分 厘日本式パーセント表記

√ と ∛ はニュートン法（反復回数は SBCD_PRECISION に基づき自動計算）による BCD ネイティブ実装です。完全桁精度で計算されます。

入力展開後結果

100-5%100×(100-5)÷10095
100×5%100×5÷1005
100÷5%100÷5×1002000

エラー

エラー原因
.tooLong数式が AZFormula.maxFormulaLength を超えた（デフォルト 200 文字。実行時に変更可能）
.negativeSqrt√ に負の値を適用した
.invalidExpression括弧の不一致・演算子の連続など

サブステップ API（テスト・カスタム処理用）

let tokens = AZFormula.tokenize("100+5%")
// ["100", "×", "(", "100", "+", "5", ")", "÷", "100"]

let rpn = AZFormula.toRPN(tokens)
// ["100", "100", "5", "+", "×", "100", "÷"]

AZCalc

BCD 十進演算と数式評価（iOS / macOS）

浮動小数点誤差のない BCD（Binary Coded Decimal）演算型 AZDecimal と、中置記法の数式文字列を評価する AZFormula を一つのパッケージで提供します。整数部・小数部それぞれ最大 30 桁（合計 60 桁）の精度を持ち、8 種の丸めモードに対応しています。

このパッケージの中核にある sbcd ライブラリは 1998 年に C/C++ で開発され、カルメモ（CalcRoll）創期から組み込まれてきた演算エンジンです。浮動小数点演算の誤差が許されない用途にご利用ください。2026 年に Swift 6 対応の Swift Package として独立公開しました。

デモ

特長

浮動小数点誤差なしBCD（2進化10進数）演算で 0.1+0.2=0.3 が保証される
60 桁の精度（初期定義）整数部・小数部それぞれ最大 30 桁。この有効桁数は、ビルドフラグ（-DSBCD_PRECISION=）で変更可能。値は偶数に限る。上限は固定 C 配列（char digit[SBCD_PRECISION+1]）のスタック消費が許される限りです。
8 種の丸めモード四捨五入・銀行家の丸め・切り上げ・切り捨てなど
数式評価エンジンShunting Yard アルゴリズムによる中置記法の解析
% 演算子の文脈依存展開100+5% → 105（増量）/ 100×5% → 5（比率）
桁区切りフォーマット3桁・4桁・インド式・任意セパレータに対応
Swift 6 Sendable 対応struct で値型。スレッドをまたいで安全に渡せる
デモアプリ付属SwiftUI デモ（iOS 17+）が同梱。Xcode ですぐに試せる

AZDecimal

基本演算

import AZDecimal

let a: AZDecimal = "1.1"
let b: AZDecimal = "2.0456"
print(a + b)   // "3.1456"
print(a - b)   // "-0.9456"
print(a * b)   // "2.25016"
print(a / b)   // "0.537815..."

var x: AZDecimal = "10"
x += "3"   // 13
x -= "5"   // 8
x *= "2"   // 16
x /= "4"   // 4

便利プロパティ

AZDecimal.zero        // AZDecimal("0")
AZDecimal.one         // AZDecimal("1")

let v: AZDecimal = "-3.14"
v.isZero              // false
v.isNegative          // true
v.abs                 // AZDecimal("3.14")

// ゼロ除算・オーバーフローは NaN を返す（Double と同じ）（2.0.0 以降）
let bad = AZDecimal("1") / AZDecimal("0")
bad.isNaN             // true
bad == bad            // false（NaN は自身とも等しくない）

※ isNaN による NaN 表現は 2.0.0 以降。2.0.0 未満では内部番兵 "-0" を返していました。

べき乗・剰余（2.0.0 以降）

AZDecimal("2").power(10)                   // 1024（整数指数は BCD 厳密）
AZDecimal("2").power(-3)                    // 0.125
AZDecimal("10").remainder(dividingBy: "3") // 1（商を0方向に切り捨て）
AZDecimal("42").integerValue               // Optional(42)
AZDecimal("3.5").integerValue              // nil

NaN・0 の負べき・0 での剰余は NaN を返します。

Codable（2.0.0 以降）

AZDecimal は Codable に準拠し、内部値文字列1本で符号化されます。計算状態・履歴・金額などの永続化が簡潔に書けます（NaN も正しく往復します）。

struct Receipt: Codable { let total: AZDecimal }

let data = try JSONEncoder().encode(Receipt(total: "1234.56"))
// {"total":"1234.56"}

let restored = try JSONDecoder().decode(Receipt.self, from: data)
restored.total   // AZDecimal("1234.56")

丸め

// フルエント設定
let config = AZDecimalConfig.default
    .digits(2)
    .rounding(.r54)
    .trailingZero(true)
    .grouping(.threes)

let result = AZDecimal("3.456").rounded(config)
// AZDecimal("3.46")

// 書式化（丸め後に呼ぶ）
let value = AZDecimal("1234567.045")
print(value.rounded(config).formatted(config))
// "1,234,567.05"

丸めモード

モード名称説明規格
.rup切り上げ絶対値の大きい方向に丸める—
.rPlus正方向丸め正の無限大方向に丸めるIEEE 754: roundTowardPositive
.r54四捨五入5以上を切り上げJIS Z 8401 規則B
.r55五捨五超入偶数丸め・銀行家の丸めJIS Z 8401 規則A · IEEE 754: roundTiesToEven
.r65五捨六入5は切り捨て・6以上で切り上げ—
.rMinus負方向丸め負の無限大方向に丸めるIEEE 754: roundTowardNegative
.truncateToDigits（2.0.0 以降）桁数で切り捨て値を decimalDigits に切り捨てるIEEE 754: roundTowardZero
.keepFull（2.0.0 以降）全桁保持丸めなし・生の値を全桁返す—

※ 2.0.0 で旧 .truncate を .truncateToDigits（値を切り捨て）と .keepFull（丸めなし）に分離しました。旧 .truncate は .keepFull 相当です。

.keepFull の用途: 連鎖計算で中間値の精度を落としたくない場合に有効です。内部値は最大精度のまま保持し、表示は formatted() の桁切りに任せます（電卓のアキュムレータなど）。

AZFormula

基本的な使い方

import AZFormula

// 文字列を返す
switch AZFormula.evaluate("(100 + 5%) × 1.08") {
case .success(let value): print(value)  // "113.4"
case .failure(let error): print(error)
}

// 設定を指定して丸める
let config = AZDecimalConfig.default.digits(2).rounding(.r54)
let result = AZFormula.evaluate("1 ÷ 3", config: config)
// .success("0.33")

AZDecimal を返す

結果をさらに演算したい場合は evaluateDecimal を使います。

if case .success(let a) = AZFormula.evaluateDecimal("10+5"),
   case .success(let b) = AZFormula.evaluateDecimal("2+1") {
    print(a * b)   // AZDecimal("45")
    print(a.isZero)  // false
}

対応演算子

演算子説明
+ - × ÷四則演算（* / も受け付ける）
^（2.0.0 以降）べき乗（整数指数のみ・右結合・× ÷ より優先）
√平方根
∛立方根
( )括弧
%パーセント（文脈依存、下表参照）
割 分 厘日本式パーセント表記

√ と ∛ はニュートン法（反復回数は SBCD_PRECISION に基づき自動計算）による BCD ネイティブ実装です。完全桁精度で計算されます。

% 演算子の展開ルール

入力展開後結果
100+5%100×(100+5)÷100105
100-5%100×(100-5)÷10095
100×5%100×5÷1005
100÷5%100÷5×1002000

パーセント記号と除数は AZFormula.percentDivisors で公開・変更できます（2.0.0 以降。既定: % → 100、割 → 10、分 → 100、厘 → 1000）。日本式表記を無効化したい場合は AZFormula.percentDivisors = ["%": "100"] のように % のみへ絞れます。AZFormula.defaultPercentDivisors で既定へ戻せます。

エラー

エラー原因
.tooLong数式が AZFormula.maxFormulaLength を超えた（デフォルト 200 文字。実行時に変更可能）
.negativeSqrt√ に負の値を適用した
.zeroDivisionゼロで除算した（2.0.0 以降）
.overflow精度桁数を超えた（オーバーフロー）（2.0.0 以降）
.unmatchedParenthesis括弧の対応が取れていない（2.0.0 以降）
.missingOperand演算子に対してオペランドが不足（2.0.0 以降）
.invalidExpression上記以外の評価できない式（不正トークンなど）

開発の歩み

2009 — はじまり

iPhone 3 が日本に登場したのを機に、アプリを作ってみようと思い立ち中古 Mac mini を購入した。Xcode 3・Objective-C・Cocoa はすべて初めてで手探り状態だったが、「自分が使いたいものを作る」という動機だけで夢中になった。

Core Data の習得も兼ねて、最初のアプリを App Store に公開した。「自己満足でも世界中で 100 人くらいにはハマるはず」という軽い気持ちだったが、その見込みは良い意味で外れることになった。

2010 — 最初の公開

最初のリリースは予想以上の反響で、モチメモは各国からダウンロードされるようになった。続けてクレメモも公開し、どちらも着実にユーザーを獲得していった。

開発を続ける中で、Picker コンポーネントは入力専用だが出力にも応用できると気づいた。店のレジからレシートが出てくるようなドラム式電卓を思い描き、実装してみると思いどおりの動きになった。海外ユーザーに特に好評だったため、後に ドラタク ⇒ CalcRoll ⇒ Calclin（カルメモ）と改名している。

電卓アプリを作ると決めたとき「浮動小数点演算は使わない」ことも決めた。1998 年に自作した BCD 演算ライブラリ（SBCD/C++）をそのまま組み込み、0.1+0.2=0.3 が保証される演算エンジンとして初版から搭載していた。このライブラリは 2026 年に AZCalc として Swift Package 化されることになる。

2011–2012 — ダイアル UI の時代

iOS でスキューモーフィズム（立体的なリアルデザイン）が流行し始めた頃、クラシックなダイアル操作をリアルに再現した独自コントロール AZDial を開発した。ノブを回す感触をスクロールとタッチで表現したもので、このコントロールを活かしたアプリをいくつか制作した。このライブラリは 2026 年に AZDial として Swift Package 化されることになる。

2013–2024 — 空白期間

アプリエンジニアとしての本業が多忙になり、個人アプリの更新が滞った約10年間。仕事でモバイル開発の経験は積み続けていたが、自分のアプリに手を入れる余裕がなかった。

2025 — Swift 6 / SwiftUI への移行

Swift 6・SwiftUI に慣れるため CalcRoll の移植を開始した。旧コードはすべて Objective-C で書かれており、現代の Swift に書き直すのは一からの再設計に近かった。新規アプリとして審査に提出したところ「類似アプリあり、新規性なし」で却下——そりゃそうだ！旧アプリのアップデートとして再提出し、無事に審査を通過した。1998 年からの SBCD ライブラリは C++ のまま組み込み利用した。

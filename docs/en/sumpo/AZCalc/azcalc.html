AZCalc

BCD decimal arithmetic and formula evaluation (iOS / macOS)

A Swift package providing AZDecimal — a floating-point-free BCD decimal type with 60-digit precision and 8 rounding modes — and AZFormula — an infix formula string evaluator built on the Shunting Yard algorithm.

The sbcd library at the heart of this package was originally written in C/C++ in 1998 and has been embedded in Calclin (CalcRoll) since its earliest days. It was born from a single conviction: a calculator app must never use floating-point arithmetic. That principle has driven over 25 years of real-world use. In 2026 it was extracted and published as a Swift 6–ready Swift Package.

Features

No floating-point errorsBCD arithmetic guarantees 0.1+0.2=0.3
60-digit precision (default definition)Up to 30 integer digits + 30 decimal digits. This significant-digit count can be changed with a build flag (-DSBCD_PRECISION=). The value must be even. The upper bound is whatever stack consumption permits for the fixed C array (char digit[SBCD_PRECISION+1]).
8 rounding modesRound half up, banker's rounding, ceiling, floor, and more
Formula evaluatorInfix parsing via Shunting Yard algorithm
Context-sensitive % operator100+5% → 105 (add) / 100×5% → 5 (ratio)
Flexible number formattingThousands, fours, Indian grouping with custom separators
Swift 6 SendableValue type struct — safe to pass across threads
SwiftUI demo app includediOS 17+ demo bundled. Open in Xcode and run instantly

AZDecimal

Basic arithmetic

import AZDecimal

let a: AZDecimal = "1.1"
let b: AZDecimal = "2.0456"
print(a + b)   // "3.1456"
print(a - b)   // "-0.9456"
print(a * b)   // "2.25016"
print(a / b)   // "0.537815..."

var x: AZDecimal = "10"
x += "3"   // 13
x -= "5"   // 8
x *= "2"   // 16
x /= "4"   // 4

Convenience properties

AZDecimal.zero        // AZDecimal("0")
AZDecimal.one         // AZDecimal("1")

let v: AZDecimal = "-3.14"
v.isZero              // false
v.isNegative          // true
v.abs                 // AZDecimal("3.14")

// Division by zero / overflow returns NaN (like Double) (2.0.0+)
let bad = AZDecimal("1") / AZDecimal("0")
bad.isNaN             // true
bad == bad            // false (NaN is not equal to itself)

Note: NaN representation via isNaN is available from 2.0.0. Earlier versions returned the internal "-0" sentinel.

Power and remainder (2.0.0+)

AZDecimal("2").power(10)                   // 1024 (BCD-exact for integer exponents)
AZDecimal("2").power(-3)                    // 0.125
AZDecimal("10").remainder(dividingBy: "3") // 1 (quotient truncated toward zero)
AZDecimal("42").integerValue               // Optional(42)
AZDecimal("3.5").integerValue              // nil

NaN, 0 raised to a negative power, and remainder by zero all return NaN.

Codable (2.0.0+)

AZDecimal conforms to Codable, encoded as a single string (its value). Persisting calculator state, history, or amounts becomes a one-liner (NaN round-trips correctly).

struct Receipt: Codable { let total: AZDecimal }

let data = try JSONEncoder().encode(Receipt(total: "1234.56"))
// {"total":"1234.56"}

let restored = try JSONDecoder().decode(Receipt.self, from: data)
restored.total   // AZDecimal("1234.56")

Rounding and formatting

// Fluent configuration
let config = AZDecimalConfig.default
    .digits(2)
    .rounding(.r54)
    .trailingZero(true)
    .grouping(.threes)

let value = AZDecimal("1234567.045")
print(value.rounded(config).formatted(config))
// "1,234,567.05"

Rounding modes

ModeNameDescriptionStandard
.rupRound UpRound away from zero—
.rPlusRound toward +∞Round toward positive infinityIEEE 754: roundTowardPositive
.r54Round Half UpRound half upJIS Z 8401 Rule B
.r55Round Half EvenRound half to even — Banker's roundingJIS Z 8401 Rule A · IEEE 754: roundTiesToEven
.r65Round Half Down5 rounds down, 6+ rounds up—
.rMinusRound toward −∞Round toward negative infinityIEEE 754: roundTowardNegative
.truncateToDigits (2.0.0+)Truncate to digitsTruncate the value to decimalDigitsIEEE 754: roundTowardZero
.keepFull (2.0.0+)Keep fullNo rounding — return the raw value at full precision—

Renamed in 2.0.0: the former .truncate is split into .truncateToDigits (truncates the value) and .keepFull (no rounding). The old .truncate behaved like .keepFull.

When to use .keepFull: preserve full precision through chained calculations. The internal value stays at maximum precision while display truncation is left to formatted() (e.g. a calculator accumulator).

AZFormula

Basic usage

import AZFormula

// Returns a String
switch AZFormula.evaluate("(100 + 5%) × 1.08") {
case .success(let value): print(value)  // "113.4"
case .failure(let error): print(error)
}

// With rounding config
let config = AZDecimalConfig.default.digits(2).rounding(.r54)
let result = AZFormula.evaluate("1 ÷ 3", config: config)
// .success("0.33")

Returns AZDecimal

Use evaluateDecimal when you need to perform further arithmetic on the result.

if case .success(let a) = AZFormula.evaluateDecimal("10+5"),
   case .success(let b) = AZFormula.evaluateDecimal("2+1") {
    print(a * b)     // AZDecimal("45")
    print(a.isZero)  // false
}

Supported operators

OperatorDescription
+ - × ÷Basic arithmetic (* / also accepted)
^ (2.0.0+)Power (integer exponent only, right-associative, binds tighter than × ÷)
√Square root
∛Cube root
( )Parentheses
%Percent — context-sensitive (see below)
割 分 厘Japanese percent notation

√ and ∛ use a native BCD Newton-Raphson implementation. Iteration count is derived automatically from SBCD_PRECISION, giving full-precision results.

Percent operator expansion rules

InputExpands toResult
100+5%100×(100+5)÷100105
100-5%100×(100-5)÷10095
100×5%100×5÷1005
100÷5%100÷5×1002000

The percent symbols and divisors are exposed via AZFormula.percentDivisors (2.0.0+; default: % → 100, 割 → 10, 分 → 100, 厘 → 1000). To disable the Japanese notation, narrow it to % only: AZFormula.percentDivisors = ["%": "100"]. Restore the default with AZFormula.defaultPercentDivisors.

Sub-step API (for testing / custom pipelines)

let tokens = AZFormula.tokenize("100+5%")
// ["100", "×", "(", "100", "+", "5", ")", "÷", "100"]

let rpn = AZFormula.toRPN(tokens)
// ["100", "100", "5", "+", "×", "100", "÷"]

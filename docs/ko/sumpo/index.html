개발 이력

2009년 — 시작

2009년, 일본에 아이폰 3가 상륙했을 때 충동적으로 중고 Mac mini를 구입하며 앱 제작을 시작했습니다. Xcode 3, Objective-C, Cocoa 모두 처음 접하는 것들이었지만, 제가 직접 원하는 것을 만든다는 동기만으로도 계속해 나갈 수 있었습니다.

Core Data를 배우면서, 아무리 개인적 취향에 치우친 앱이라도 전 세계 어딘가에는 팬 100명쯤은 있지 않을까 싶어 App Store에 출시하기 시작했습니다. 하지만 그것은 지나치게 소극적인 예상이었습니다.

2010년 — 첫 출시

첫 출시작들은 예상보다 훨씬 큰 반응을 얻었습니다. 모치메모(Mochi Memo)는 여러 나라 사용자들에게 다운로드되었고, 크레딧 메모(Credit Memo) 역시 그 곁에서 꾸준히 이용자층을 넓혀 갔습니다.

개발 중에 문득, 보통은 선택 입력에 쓰이는 피커(picker)가 출력용으로도 쓰일 수 있겠다는 생각이 들었습니다. 금전등록기에서 영수증이 굴러 나오는 모습을 떠올리며 드럼식 계산기를 만들었습니다. 이름은 몇 차례 바뀌었습니다. Drataku ⇒ CalcRoll ⇒ Calclin 순이었습니다.

계산기 앱을 만들기로 하면서 한 가지도 함께 결정했습니다. 부동소수점 연산은 쓰지 않는다는 것이었습니다. 1998년에 직접 작성한 BCD 연산 라이브러리(SBCD/C++)를 그대로 내장해, 첫날부터 0.1+0.2=0.3이 성립하도록 만들었습니다. 이 라이브러리는 훗날 2026년에 AZCalc Swift Package로 공개됩니다.

2013–2024년 — 조용했던 10년

앱 엔지니어로서 본업이 바빠지면서 개인 프로젝트에 쏟을 여유가 거의 없었습니다. 업무를 통해 모바일 개발 경험은 계속 쌓였지만, 정작 제 앱을 돌볼 여력은 없었습니다.

2025년 — Swift 6 / SwiftUI로의 전환

Swift 6와 SwiftUI를 익히기 위해 CalcRoll을 이식했습니다. 기존 코드베이스는 전부 Objective-C로 작성되어 있었기 때문에, 최신 Swift로 다시 쓰는 작업은 사실상 처음부터 새로 설계하는 것이나 다름없었습니다. 처음에는 신규 앱으로 제출했지만 "유사한 앱이 이미 존재하며 참신함이 부족하다"는 이유로 애플에게 반려당했습니다. 틀린 말은 아니었기에 기존 앱의 업데이트로 다시 제출해 심사를 통과했습니다. 1998년에 작성한 SBCD 라이브러리는 C++ 코드 그대로 계승해 그대로 내장했습니다.
